# Demo Notes

## Product form

A small developer workflow demo for turning one focused GitHub issue into a clear review package.

## MVP screens

1. Issue intake: title, body, labels, and acceptance criteria.
2. Repository map: folders, package manager, test command, and important files.
3. Selected files: the files chosen for the change, with short reasons.
4. Plan: a small change plan before implementation.
5. Result: what changed.
6. Test output: command and short result.
7. Review summary: what a human reviewer should check.

## Demo flow

1. Select one small GitHub issue.
2. Show the accepted criteria.
3. Show selected files.
4. Show the plan.
5. Show the result.
6. Show test output.
7. Show the final review summary.

## Good demo points

- The scope stays small.
- File selection is explained.
- The result is easy to review.
- The demo is useful even before full automation exists.
